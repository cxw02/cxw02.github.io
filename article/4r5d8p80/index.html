一个专门发送私密且安全的便条应用-enclosed

约 810 字大约 3 分钟

docker文件传输

2025-09-10

Enclosed：一款简约而强大的网络应用程序。它通过端到端加密，确保服务器和存储端对内容一无所知；用户可以设置密码、有效期（TTL），甚至让便条在被阅读后立即自毁，从而大幅提升信息的安全性。

本文将带大家使用 Docker 快速部署 Enclosed，让你轻松拥有一个属于自己的安全便条服务，在需要分享敏感信息时，再也不用担心安全问题。

项目简介

Enclosed是一个简约的网络应用程序，旨在发送私密且安全的笔记。

所有笔记均采用端到端加密，确保服务器和存储对内容一无所知。用户可以设置密码、定义有效期 (TTL)，并选择在阅读后将笔记自毁。

项目特征

端到端加密：您的笔记在客户端加密，使用 AES-GCM 和使用 PBKDF2 派生的 256 位密钥。
文件附件：与您的笔记安全地共享文件。
零知识：服务器无法访问笔记或文件的内容。
可配置的安全选项：设置密码、到期时间，并选择阅读笔记后自毁。
简约的用户界面：简单直观的用户界面，可快速共享笔记。
i18n 支持：提供多种语言。
身份验证：可选电子邮件/密码身份验证来创建笔记。
黑暗模式：适合深夜分享笔记的黑暗主题。
响应式设计：适用于所有设备，从台式机到手机。
开源：源代码可在 Apache 2.0 许可下使用。
自托管：运行 Enclosed 实例以进行私人笔记共享。
CLI：用于从终端创建笔记的命令行界面。
对环境的影响非常小：该应用程序和文档在 websitecarbon.com 上被评为 A+（参见此处）。

使用docker部署

docker cli推荐

docker run -d --restart=unless-stopped ^
  --name enclosed ^
  -p 8787:8787 ^
  -v enclosed-data:/app/.data ^
  corentinth/enclosed:1.16.0

命令说明

  • -d：以分离模式（在后台）运行容器。
  • --restart unless-stopped：将容器配置为始终重新启动，除非它被显式停止。
  • --name enclosed：将容器命名为“enclosed”，以便于识别。
  • -p 8787:8787：将主机上的端口 8787 映射到容器中的端口 8787，使应用程序可以在本地计算机上访问。
  • -v enclosed-data:/app/.data：将本地目录映射到容器的数据目录，确保数据持久存储在主机上。
  • corentinth/enclosed:1.16.0：指定要使用的 Docker 映像。

docker compose

确保您的系统上安装了 Docker 和 Docker Compose。您可以按照Docker Compose 官方网站上的说明安装 Docker Compose。

docker-compose.yml
name: enclosed
services:
  enclosed:
    container_name: enclosed
    image: corentinth/enclosed:1.16.0
    ports:
      - 8787:8787
    volumes:
      - enclosed-data:/app/.data
    restart: unless-stopped

volumes:
  enclosed-data:
    driver: local

启动服务

要使用 Docker Compose 启动 Enclosed ，请运行：

docker compose up -d

停止服务

要停止Enclosed服务，请运行：

docker compose down

查看日志

若要查看 Enclosed 服务的日志，请运行：

docker-compose logs -f enclosed
